#### git merge와는 다른 rebase

리베이스 rebase

rebase는 merge와 동일하게 하나의 브랜치를 다른 브랜치로 병합하는 기능

**단 차이점은,**

```
두 브랜치의 결과를 병합하는 merge와 달리

rebase는 한 브랜치에서 이루어진 작업 히스토리를 다른 브랜치에 차례대로 적용시킵니다.
```

**왜 사용하는가 ?**

```
둘의 결과물은 완전히 동일하나, rebase는 병합 히스토리를 더욱 깔끔하게 남겨줌

공동 작업에서 깔끔한 히스토리를 계속 유지해주고 싶다면 REBASE를 적극
```
